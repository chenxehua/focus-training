## 1. 产品概述

### 1.1 产品定位

**产品名称**：儿童专注力训练工具

**一句话描述**：基于神经科学原理的家庭场景专注力训练工具

**核心价值**：
- 为4-12岁儿童提供科学的专注力训练游戏
- 帮助家长追踪和了解孩子的专注力发展
- 以线下机构1/10的价格，提供可量化的训练效果

### 1.2 产品定位声明

> ⚠️ **重要声明**：本产品是专注力**训练工具**，不是医疗诊断或治疗手段。本产品不能替代专业医疗评估或治疗。如有注意力相关困扰，建议咨询专业医生。

### 1.3 产品目标

| 目标类型 | 目标描述 | 衡量指标 |
|----------|----------|----------|
| 用户目标 | 帮助儿童提升专注力 | 专注时长提升 |
| 商业目标 | 建立可持续的商业模式 | 付费转化率≥15% |
| 社会目标 | 减轻家长育儿焦虑 | 用户满意度≥85% |

### 1.4 产品边界

**做（本期）：**
- 专注力游戏训练
- 专注时长追踪
- 家长报告
- 分龄难度系统
- 正向激励体系

**不做（明确）：**
- ADHD诊断
- 医疗治疗
- 处方建议
- 线下服务
- 社交功能（初期）

## 2. 目标用户画像

### 2.1 主用户：儿童家长

#### 核心用户画像

| 属性 | 描述 |
|------|------|
| 年龄 | 28-42岁 |
| 学历 | 大专及以上 |
| 职业 | 在职父母为主 |
| 地域 | 一二线城市为主 |
| 收入 | 月入1-3万 |
| 核心诉求 | 解决孩子注意力问题、提升学习效率 |

#### 用户分群

| 分群 | 特征 | 痛点 | 需求 |
|------|------|------|------|
| **轻度关注** | 孩子有轻微走神 | 写作业慢 | 日常训练工具 |
| **中度关注** | 老师反映注意力问题 | 影响成绩 | 系统化训练方案 |
| **重度关注** | 疑似/已确诊ADHD | 焦虑、无助 | 专业辅助+家长指导 |

#### 家长用户故事

**典型场景1：张妈妈（32岁，深圳）**
> "孩子上二年级，写作业5分钟就要喝水、上厕所，一小时作业能磨蹭到半夜。报了线下班太贵，而且接送麻烦。希望有个能在家做的训练工具，孩子不抗拒最好。"

**典型场景2：李爸爸（38岁，上海）**
> "儿子被老师建议去检查，怀疑有多动倾向。但我不想给孩子贴标签，也不想一上来就吃药。想先试试有没有其他方法，看看训练能不能有效果。"

**典型场景3：王妈妈（35岁，北京）**
> "孩子确诊ADHD两年，在吃药。想找个能配合治疗的辅助工具，平时在家可以做一些训练。最好能看到训练效果，能跟医生沟通就更好了。"

### 2.2 直接用户：儿童

#### 儿童用户画像

| 属性 | 描述 |
|------|------|
| 年龄 | 4-12岁 |
| 分段 | 4-6岁（幼儿）、7-9岁（小学低）、10-12岁（小学高） |
| 核心诉求 | 好玩、有趣、不无聊 |
| 使用特点 | 自主使用或家长陪伴使用 |

#### 儿童用户分层需求

| 年龄段 | 认知特点 | 训练重点 | 设计偏好 |
|--------|----------|----------|----------|
| 4-6岁 | 形象思维为主 | 观察力、记忆力 | 卡通形象、简单操作 |
| 7-9岁 | 开始抽象思维 | 持续注意力、抗干扰 | 任务挑战、即时反馈 |
| 10-12岁 | 逻辑思维发展 | 执行功能、自控力 | 成就系统、成长感 |

## 5. 用户故事与验收标准

### 5.1 用户故事格式

```
用户故事格式：
作为 [角色]
我想要 [功能]
以便 [价值]

验收标准（Acceptance Criteria）：
1. Given [前置条件]
   When [操作]
   Then [预期结果]
```

### 5.2 核心用户故事

#### 用户故事1：家长注册并添加孩子

```
作为 家长用户
我想要 注册账号并添加孩子信息
以便 开始使用训练服务

验收标准：
1. Given 用户打开小程序
   When 选择"微信登录"并授权
   Then 完成登录并进入引导页
   
2. Given 用户完成登录
   When 输入孩子姓名、年龄、性别
   Then 保存信息并进入首页

3. Given 家长有多个孩子
   When 点击"添加孩子"
   Then 支持添加第2-3个孩子

4. Given 孩子年龄为8岁
   When 系统自动设置难度为"7-9岁"档
   Then 后续训练基于该难度推荐
```

#### 用户故事2：完成一次训练

```
作为 儿童用户（家长辅助）
我想要 完成一次专注力训练游戏
以便 提升专注力并获得奖励

验收标准：
1. Given 儿童进入首页
   When 点击"今日任务"
   Then 展示当日推荐训练内容
   
2. Given 儿童选择"舒尔特方格"
   When 点击"开始训练"
   Then 进入游戏并开始计时
   
3. Given 游戏进行中
   When 儿童正确点击数字
   Then 显示正确反馈并播放音效
   
4. Given 游戏进行中
   When 儿童点击错误
   Then 显示错误反馈并扣分
   
5. Given 游戏结束
   When 显示成绩（用时、准确率、得分）
   Then 同时发放奖励（经验值、徽章碎片）

6. Given 训练完成
   When 系统记录训练数据
   Then 家长可在报告中查看
```

#### 用户故事3：查看训练报告

```
作为 家长用户
我想要 查看孩子的训练报告
以便 了解训练效果

验收标准：
1. Given 家长进入报告页
   When 查看"今日报告"
   Then 展示：训练时长、次数、专注度、进步亮点
   
2. Given 家长查看"周报告"
   When 切换到周视图
   Then 展示：本周趋势图、与上周对比、能力分布
   
3. Given 报告加载中
   When 网络较慢
   Then 显示加载动画，不超过5秒

4. Given 家长看到进步数据
   When 点击"训练建议"
   Then 展示针对性的训练建议

5. Given 数据存在异常
   When 系统检测到训练数据异常
   Then 提示家长关注并建议调整
```

#### 用户故事4：订阅会员

```
作为 家长用户
我想要 订阅会员
以便 解锁全部训练内容

验收标准：
1. Given 家长查看会员页
   When 选择"季付249元"
   Then 展示订阅确认页
   
2. Given 家长点击"立即订阅"
   When 完成微信支付
   Then 订阅成功，解锁全部功能
   
3. Given 订阅成功
   When 弹窗提示会员权益
   Then 同时推送通知确认

4. Given 会员即将到期（剩余3天）
   When 家长打开小程序
   Then 显示续费提醒

5. Given 会员到期
   When 未续费
   Then 关闭付费功能，免费功能可用
```

## 8. 合规说明

### 8.1 产品定位声明

> ⚠️ **产品定位声明**
> 
> 本产品名称：**儿童专注力训练工具**
> 
> 产品定位：**专注力训练辅助工具**
> 
> 本产品**不是**：
> - 医疗诊断手段
> - 疾病治疗方案
> - 专业医疗建议的替代
> 
> 本产品**是**：
> - 基于神经科学原理设计的训练工具
> - 帮助儿童提升专注力的辅助手段
> - 辅助家长进行家庭训练的参考

### 8.2 合规表述规范

#### ✅ 合规表述示例

| 场景 | 合规表述 |
|------|----------|
| 产品介绍 | "专注力训练"、"注意力提升训练" |
| 效果描述 | "帮助养成良好学习习惯"、"提升专注时长" |
| 家长沟通 | "辅助家长进行家庭训练"、"如有疑虑请咨询专业医生" |
| 报告内容 | "专注力发展参考"、"训练数据仅供参考" |

#### ❌ 禁止表述示例

| 类型 | 禁止表述 |
|------|----------|
| 诊断相关 | "判断/诊断ADHD"、"评估是否多动" |

| 医疗暗示 | "临床验证疗效"、"替代药物治疗" |

### 8.4 建议就医场景

当产品发现以下情况时，应明确建议用户寻求专业医疗帮助：

| 情况 | 提示内容 |
|------|----------|
| 用户自述已确诊ADHD | 建议配合医疗治疗使用，本产品为辅助训练 |
| 训练效果持续不理想 | 建议进行专业评估 |
| 用户主动询问是否有多动症 | 建议就医检查，不提供判断 |
| 存在明显行为异常 | 建议寻求专业帮助 |
