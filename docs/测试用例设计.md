## 1. 测试策略与范围

### 1.1 测试策略

| 测试类型 | 策略 | 占比 |
|----------|------|------|
| 功能测试 | 全面覆盖核心流程 | 50% |
| 边界测试 | 覆盖极端情况和异常输入 | 15% |
| 安全测试 | 重点关注儿童数据保护 | 15% |
| 性能测试 | 确保流畅度和响应速度 | 10% |
| 合规测试 | 确保无医疗诊断暗示 | 10% |

### 1.2 测试范围

#### 纳入测试范围

| 模块 | 测试内容 |
|------|----------|
| 用户系统 | 注册、登录、家长-儿童绑定 |
| 训练游戏 | 舒尔特方格、听声辨数、图案记忆 |
| 计时系统 | 专注时长记录、精确度 |
| 报告系统 | 日报、周报生成 |
| 激励系统 | 成就解锁、经验值、等级 |
| 会员系统 | 订阅、支付、权益 |

#### 暂不测试范围

| 模块 | 原因 |
|------|------|
| 微信小程序审核流程 | 不可控因素 |
| 第三方支付回调 | 依赖外部系统 |
| AI个性化推荐 | V2功能 |

### 1.3 测试环境

| 环境 | 用途 | 配置 |
|------|------|------|
| 开发环境 | 日常开发自测 | 本地搭建 |
| 测试环境 | 功能测试 | 与生产环境1:1 |
| 预发布环境 | 回归测试 | 生产数据脱敏 |
