## 1. 项目背景与目标

### 1.1 项目背景

#### 市场需求驱动

根据市场调研数据，中国儿童专注力训练市场规模在2025年预计达到**167-280亿元**，年复合增长率达**17%-25%**。然而市场渗透率仅**5%-8%**，说明绝大多数目标用户的需求尚未被有效满足。

| 市场指标 | 数据 |
|----------|------|
| 市场规模（2025E） | 167-280亿元 |
| 年复合增长率 | 17%-25% |
| 目标用户规模 | 1.2亿（3-12岁儿童） |
| ADHD患病率 | 6.4%（约2300万潜在用户） |
| 市场渗透率 | 5%-8% |

#### 用户痛点验证

- **52.32%** 的家长因孩子注意力不集中而崩溃
- **38.5%** 学龄儿童专注力不达标
- 每3个6-12岁儿童中就有1个存在上课走神、作业拖拉问题

#### 现有方案差距

| 方案类型 | 价格区间 | 痛点 |
|----------|----------|------|
| 线下机构 | 5000-20000元/疗程 | 价格高昂 |
| 现有App | 98-199元/月 | 专业性不足、效果不持续 |
| 自己摸索 | 0元 | 无效、缺乏科学指导 |

#### 价格空白带

```
线下机构(5000-20000元)
        ↓
[价格空白带] ← 我们的切入区间 (99-149元/月)
        ↓
现有App(<1000元/年)
```

### 1.2 项目目标

#### 核心目标

为4-12岁儿童提供基于神经科学原理的家庭场景专注力训练工具，让家长以**线下机构1/10的价格**，获得科学化、个性化、效果可追踪的训练方案。

#### 业务目标

| 指标 | 目标值 | 说明 |
|------|--------|------|
| 首年付费用户 | 10,000+ | 转化率15% |
| 次月留存率 | ≥40% | 参考验证标准 |
| 7日留存率 | ≥15% | 参考验证标准 |
| 用户满意度 | ≥85% | NPS评分 |

## 2. 建设范围与边界

### 2.1 核心功能范围

#### 服务对象

- **主要用户**：4-12岁儿童家长（6-9岁小学低年级为核心）
- **直接用户**：4-12岁儿童（产品使用者）

#### 功能边界（本期）

| 模块 | 包含功能 | 优先级 |
|------|----------|--------|
| 训练游戏 | 3-5款专注力训练游戏 | P0 |
| 专注追踪 | 训练时长记录与分析 | P0 |
| 家长报告 | 周期性训练报告 | P0 |
| 分龄系统 | 年龄分段难度调节 | P1 |
| 训练计划 | 21天训练计划 | P1 |
| 激励体系 | 成就徽章系统 | P1 |
| 会员服务 | 订阅制会员管理 | P1 |

#### 功能边界（明确不做）

| 排除项 | 原因 |
|--------|------|
| ADHD诊断 | 超出"训练工具"定位，涉及医疗行为 |
| 处方治疗 | 需医疗机构资质 |
| 线下服务 | 运营复杂度超出MVP范围 |
| 社交功能（初期） | 儿童用户安全考量 |
| 脑电设备对接 | 硬件成本高、用户门槛高 |

| 包含 | 不包含 |

| 云端数据存储 | 离线模式 |
| 订阅支付 | 线下支付 |
| 家长端管理 | 儿童端完全自主 |

### 4.2 前端技术选型

#### 微信小程序（首选）

| 技术栈 | 选择 | 理由 |
|--------|------|------|
| 框架 | uni-app / 原生开发 | uni-app跨端能力强，原生性能好 |
| 状态管理 | Redux / MobX | 视项目复杂度选择 |
| UI组件 | Vant Weapp / ColorUI | 成熟组件库 |
| 游戏引擎 | Phaser.js / Cocos | H5游戏开发首选 |
| 动画 | Lottie / CSS动画 | 轻量化选择 |

#### 推荐方案：uni-app + Phaser.js

**理由**：
1. **开发效率高**：一套代码多端运行
2. **游戏能力强**：Phaser.js是成熟H5游戏引擎
3. **生态完善**：插件市场丰富
4. **学习成本低**：Vue语法上手快

### 4.3 后端技术选型

#### 个人开发者友好方案

| 组件 | 选择 | 理由 |
|------|------|------|
| 语言 | Node.js / Python | 生态丰富、学习曲线平缓 |
| 框架 | Express / FastAPI | 轻量、灵活 |
| 数据库 | MySQL 8.0 | 稳定、文档完善 |
| 缓存 | Redis | 高性能、支持多种数据结构 |
| 云服务 | 阿里云/腾讯云轻量应用 | 成本低、运维简单 |

#### 推荐方案：Node.js + Express + MySQL

**理由**：
1. **全栈统一**：前端JS/后端JS统一技术栈
2. **异步优势**：处理高并发训练数据写入
3. **部署简单**：Serverless或轻量服务器即可
4. **社区活跃**：遇到问题容易找到解决方案

## 6. 风险管理与应对

### 6.1 风险识别与评估

| 风险类别 | 风险项 | 等级 | 发生概率 |
|----------|--------|------|----------|
| **合规风险** | ADHD领域合规红线 | 🔴高 | 中 |
| **产品风险** | 效果难以短期显现 | 🟡中 | 高 |
| **产品风险** | 儿童用户依从性低 | 🔴高 | 高 |
| **市场风险** | 竞品同质化 | 🟡中 | 高 |
| **技术风险** | 游戏性能问题 | 🟡中 | 中 |
| **运营风险** | 家长期望过高 | 🟡中 | 高 |

### 6.2 合规风险管理（重点）

#### ADHD合规红线

**绝对禁止事项：**

| 红线 | 违规后果 | 应对措施 |
|------|----------|----------|
| ❌ 声称能"诊断"ADHD | 违反医疗广告法 | 明确声明"非诊断工具" |
| ❌ 声称能"治疗"ADHD | 无医疗资质 | 定位为"训练辅助" |
| ❌ 使用"治愈"、"康复"等词汇 | 虚假宣传 | 使用"训练"、"提升" |
| ❌ 提供处方建议 | 超出服务范围 | 仅提供一般性建议 |
| ❌ 暗示产品替代医疗 | 法律风险 | 明确建议"就医"场景 |

**合规表述规范：**

```
✅ 正确表述：
   - "专注力训练" 而非 "ADHD治疗"
   - "提升注意力" 而非 "治愈多动症"
   - "辅助家长" 而非 "替代医疗"
   - "科学训练方法" 而非 "临床验证疗效"

❌ 禁止表述：
   - "改善/治疗/治愈多动症"
   - "解决注意力缺陷"
   - "替代药物治疗"
   - "专业医疗建议"
```

#### 合规实现机制

1. **产品设计层面**
   - 启动页明确声明"本产品为训练工具，非医疗诊断或治疗手段"
   - 所有涉及ADHD描述的页面添加"如有疑虑请咨询专业医生"提示
   - 报告内容避免使用疾病诊断术语

2. **内容审核层面**
   - 建立内容审核机制，所有文案经合规审查
   - 禁止出现任何暗示医疗效果的表述
   - 家长内容中强调"辅助"定位

3. **法律保护层面**
   - 用户协议中明确产品定位
   - 免责声明覆盖医疗建议边界
   - 隐私政策符合《个人信息保护法》要求

### 6.3 效果风险管理

#### 效果可见性策略

| 策略 | 实施方式 |
|------|----------|
| 过程指标优先 | 聚焦"训练完成率"而非"成绩提升" |

## 7. 产品定位说明

### 7.1 核心定位

**产品定位：专注力训练工具（Training Tool）**

> 本产品是基于神经科学原理设计的专注力**训练工具**，帮助儿童通过科学的训练方法提升注意力相关能力。**不是**医疗诊断或治疗手段。

### 7.2 用户沟通规范

#### 对家长的沟通原则

1. **诚实透明**
   - 明确告知产品的功能边界
   - 诚实说明预期效果和周期
   - 不夸大、不承诺

2. **专业引导**
   - 解释专注力提升的科学原理
   - 提供循证训练方法
   - 建议专业就医场景

3. **协同定位**
   - 产品是家长的"助手"而非"替代"
   - 强调家庭训练的重要性
   - 提供家长指导内容

#### 禁止使用的表述

| 类别 | 禁止表述 | 建议替代 |
|------|----------|----------|
| 疾病相关 | "改善多动症" | "提升专注力" |
| 医疗相关 | "治疗" | "训练" |
| 效果承诺 | "彻底解决" | "帮助改善" |
| 诊断相关 | "评估是否有多动症" | "了解专注力水平" |

### 7.3 建议就医场景

当产品发现以下情况时，应建议用户寻求专业医疗帮助：

| 情况 | 处理方式 |
|------|----------|
| 用户自述已确诊ADHD | 建议配合医疗治疗使用 |
| 训练效果持续不理想 | 建议专业评估 |
| 用户主动询问诊断 | 建议就医，不提供判断 |
| 存在明显行为异常 | 建议寻求专业帮助 |
