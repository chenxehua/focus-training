### 2.3 后端详细任务

#### API-001: 获取测评状态
```typescript
// 输入: childId
// 输出: 
{
  hasCompletedInitial: boolean,
  lastAssessmentDate: string,
  stages: {
    questionnaire: { completed: boolean, answers: number },
    games: { completed: boolean, games: number },
    report: { generated: boolean }
  },
  nextStep: 'questionnaire' | 'games' | 'report' | null
}
```

#### API-002: 开始测评
```typescript
// 输入: { childId: number, type: 'initial' | 'followup' }
// 输出:
{
  assessmentId: string,
  childInfo: { name, age, ageGroup },
  estimatedDuration: number,
  totalSteps: number,
  consentRequired: boolean
}
```

#### API-003: 获取问卷
```typescript
// 输入: assessmentId
// 输出:
{
  questions: Question[],
  totalQuestions: number,
  currentQuestion: number,
  estimatedTime: number
}
```

#### API-004: 提交问卷答案
```typescript
// 输入: { answers: Answer[], completed: boolean }
// 输出:
{
  saved: number,
  nextQuestion: number,
  progress: { current: number, total: number }
}
```

#### API-005: 获取测评游戏列表
```typescript
// 输入: assessmentId
// 输出:
{
  games: Game[],
  totalGames: number,
  currentGame: number,
  estimatedTime: number
}
```

#### API-006: 提交游戏结果
```typescript
// 输入: { gameId: string, result: GameResult }
// 输出:
{
  saved: boolean,
  performance: { score, percentile, rating },
  nextGame: string,
  progress: { completed: number, total: number }
}
```

#### API-007: 生成报告
```typescript
// 输入: assessmentId
// 输出:
{
  reportId: string,
  overallScore: number,
  percentileRank: number,
  rating: string,
  dimensions: Dimension[],
  strengths: string[],
  weaknesses: string[],
  trainingPlan: TrainingPlan
}
```

## 六、开发顺序

```
阶段一: 数据库 (1-2天)
  └─ 确保所有表和数据就绪

阶段二: 后端API (3-5天)
  ├─ 测评状态管理
  ├─ 问卷API
  ├─ 游戏结果API
  └─ 报告生成

阶段三: 前端页面 (5-7天)
  ├─ 基础页面框架
  ├─ 问卷测评流程
  └─ 游戏集成

阶段四: 测试 (3-5天)
  ├─ 单元测试
  ├─ 集成测试
  └─ E2E测试
```
