## 1. 设计概述

### 1.1 数据库选型

| 组件 | 选择 | 版本 | 说明 |
|------|------|------|------|
| **关系型数据库** | MySQL | 8.0 | 主数据存储 |
| **缓存数据库** | Redis | 6.0 | 会话、排行榜 |
| **文档数据库** | MongoDB | 5.0 | 可选：日志、分析数据 |

### 1.2 设计原则

| 原则 | 说明 |
|------|------|
| 规范化 | 遵循3NF，减少数据冗余 |
| 可扩展 | 预留扩展字段 |
| 安全性 | 敏感数据加密存储 |
| 性能 | 合理索引，按需分表 |

### 1.3 命名规范

| 规范 | 示例 |
|------|------|
| 表名 | 小写下划线，如 `user_info` |
| 字段名 | 小写下划线，如 `user_id` |
| 主键 | `id` |
| 外键 | `{table}_id` |
| 索引 | `idx_{field}` |
| 创建时间 | `created_at` |
| 更新时间 | `updated_at` |

### 2.2 核心实体说明

| 实体 | 说明 | 主要关系 |
|------|------|----------|
| user | 家长用户 | 1:N关联children |
| child | 儿童用户 | N:1关联user, 1:N训练记录 |
| parent_child | 亲子关系 | user与child的关联表 |
| game | 训练游戏 | 1:N训练记录 |
| training_record | 训练记录 | 核心业务数据 |
| training_plan | 训练计划 | 1:N每日任务 |
| achievement | 成就 | 1:N儿童解锁记录 |
| child_achievement | 儿童成就 | child与achievement关联 |
| membership | 会员 | 1:1关联user |
| focus_report | 专注力报告 | 定期生成 |

## 3. 核心数据表设计

### 3.1 用户相关表

#### 3.1.1 用户表（user）

```sql
-- 用户表：存储家长用户信息
CREATE TABLE `user` (
    `id` BIGINT UNSIGNED NOT NULL AUTO_INCREMENT COMMENT '用户ID',
    `phone` VARCHAR(20) NOT NULL COMMENT '手机号',
    `nickname` VARCHAR(50) DEFAULT NULL COMMENT '昵称',
    `avatar` VARCHAR(255) DEFAULT NULL COMMENT '头像URL',
    `role` TINYINT NOT NULL DEFAULT 1 COMMENT '角色: 1-家长',
    `status` TINYINT NOT NULL DEFAULT 1 COMMENT '状态: 1-正常, 0-禁用',
    `last_login_at` DATETIME DEFAULT NULL COMMENT '最后登录时间',
    `last_login_ip` VARCHAR(50) DEFAULT NULL COMMENT '最后登录IP',
    `wx_openid` VARCHAR(100) DEFAULT NULL COMMENT '微信openid',
    `wx_unionid` VARCHAR(100) DEFAULT NULL COMMENT '微信unionid',
    `created_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    `updated_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    `deleted_at` DATETIME DEFAULT NULL COMMENT '删除时间',
    PRIMARY KEY (`id`),
    UNIQUE KEY `uk_phone` (`phone`),
    KEY `idx_wx_openid` (`wx_openid`),
    KEY `idx_wx_unionid` (`wx_unionid`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='用户表';
```

#### 3.1.2 儿童表（child）

```sql
-- 儿童表：存储儿童用户信息
CREATE TABLE `child` (
    `id` BIGINT UNSIGNED NOT NULL AUTO_INCREMENT COMMENT '儿童ID',
    `user_id` BIGINT UNSIGNED NOT NULL COMMENT '所属家长ID',
    `name` VARCHAR(50) NOT NULL COMMENT '姓名',
    `age` TINYINT UNSIGNED NOT NULL COMMENT '年龄',
    `gender` TINYINT DEFAULT 0 COMMENT '性别: 0-未知, 1-男, 2-女',
    `avatar` VARCHAR(255) DEFAULT NULL COMMENT '头像URL',
    `age_group` VARCHAR(20) NOT NULL COMMENT '年龄段: 4-6, 7-9, 10-12',
    `level` INT NOT NULL DEFAULT 1 COMMENT '当前等级',
    `experience` INT NOT NULL DEFAULT 0 COMMENT '经验值',
    `total_training_count` INT NOT NULL DEFAULT 0 COMMENT '累计训练次数',
    `total_training_duration` INT NOT NULL DEFAULT 0 COMMENT '累计训练时长(秒)',
    `status` TINYINT NOT NULL DEFAULT 1 COMMENT '状态: 1-正常, 0-禁用',
    `created_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    `updated_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    `deleted_at` DATETIME DEFAULT NULL COMMENT '删除时间',
    PRIMARY KEY (`id`),
    KEY `idx_user_id` (`user_id`),
    KEY `idx_age_group` (`age_group`),
    CONSTRAINT `fk_child_user` FOREIGN KEY (`user_id`) REFERENCES `user` (`id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='儿童表';
```

#### 3.1.3 亲子关系表（parent_child）

```sql
-- 亲子关系表：管理家长与儿童的关联
CREATE TABLE `parent_child` (
    `id` BIGINT UNSIGNED NOT NULL AUTO_INCREMENT COMMENT '关系ID',
    `user_id` BIGINT UNSIGNED NOT NULL COMMENT '家长ID',
    `child_id` BIGINT UNSIGNED NOT NULL COMMENT '儿童ID',
    `relation_type` VARCHAR(20) DEFAULT 'parent' COMMENT '关系类型: parent-父母, guardian-监护人',
    `is_active` TINYINT NOT NULL DEFAULT 1 COMMENT '是否激活: 1-是, 0-否',
    `created_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    PRIMARY KEY (`id`),
    UNIQUE KEY `uk_user_child` (`user_id`, `child_id`),
    KEY `idx_child_id` (`child_id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='亲子关系表';
```

### 3.2 游戏相关表

#### 3.2.1 游戏表（game）

```sql
-- 游戏表：存储训练游戏配置
CREATE TABLE `game` (
    `id` BIGINT UNSIGNED NOT NULL AUTO_INCREMENT COMMENT '游戏ID',
    `game_code` VARCHAR(50) NOT NULL COMMENT '游戏代码: schulte, audio_number, pattern_memory',
    `game_name` VARCHAR(100) NOT NULL COMMENT '游戏名称',
    `game_type` VARCHAR(50) NOT NULL COMMENT '游戏类型: visual, auditory, memory, coordination',
    `description` TEXT COMMENT '游戏描述',
    `instruction` TEXT COMMENT '游戏说明',
    `icon_url` VARCHAR(255) DEFAULT NULL COMMENT '图标URL',
    `thumbnail_url` VARCHAR(255) DEFAULT NULL COMMENT '缩略图URL',
    `difficulty_levels` JSON COMMENT '难度配置JSON',
    `target_age_group` JSON COMMENT '目标年龄段JSON',
    `training_focus` JSON COMMENT '训练重点JSON',
    `base_experience` INT NOT NULL DEFAULT 10 COMMENT '基础经验值',
    `is_active` TINYINT NOT NULL DEFAULT 1 COMMENT '是否启用: 1-是, 0-否',
    `is_vip` TINYINT NOT NULL DEFAULT 0 COMMENT '是否VIP专属: 1-是, 0-否',
    `sort_order` INT NOT NULL DEFAULT 0 COMMENT '排序',
    `created_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    `updated_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    PRIMARY KEY (`id`),
    UNIQUE KEY `uk_game_code` (`game_code`),
    KEY `idx_game_type` (`game_type`),
    KEY `idx_is_active` (`is_active`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='游戏表';

-- 示例难度配置
-- {"level_1": {"grid_size": 3, "time_limit": 60}, "level_2": {"grid_size": 5, "time_limit": 90}}
```

#### 3.2.2 训练记录表（training_record）

```sql
-- 训练记录表：核心业务表，存储每次训练数据
CREATE TABLE `training_record` (
    `id` BIGINT UNSIGNED NOT NULL AUTO_INCREMENT COMMENT '记录ID',
    `child_id` BIGINT UNSIGNED NOT NULL COMMENT '儿童ID',
    `game_id` BIGINT UNSIGNED NOT NULL COMMENT '游戏ID',
    `plan_id` BIGINT UNSIGNED DEFAULT NULL COMMENT '训练计划ID(nullable)',
    `plan_day` INT DEFAULT NULL COMMENT '计划第几天(nullable)',
    `training_date` DATE NOT NULL COMMENT '训练日期',
    `duration_seconds` INT NOT NULL COMMENT '训练时长(秒)',
    `accuracy` DECIMAL(5,2) NOT NULL COMMENT '准确率(%)',
    `score` INT NOT NULL COMMENT '得分',
    `focus_score` DECIMAL(5,2) NOT NULL COMMENT '专注度评分(%)',
    `difficulty_level` INT NOT NULL DEFAULT 1 COMMENT '难度等级',
    `correct_count` INT NOT NULL DEFAULT 0 COMMENT '正确次数',
    `error_count` INT NOT NULL DEFAULT 0 COMMENT '错误次数',
    `max_combo` INT NOT NULL DEFAULT 0 COMMENT '最大连击数',
    `game_config` JSON COMMENT '游戏配置JSON',
    `result_data` JSON COMMENT '详细结果JSON',
    `experience_earned` INT NOT NULL DEFAULT 0 COMMENT '获得经验值',
    `achievements_unlocked` JSON COMMENT '解锁成就JSON',
    `device_info` VARCHAR(255) DEFAULT NULL COMMENT '设备信息',
    `app_version` VARCHAR(20) DEFAULT NULL COMMENT 'App版本',
    `created_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    PRIMARY KEY (`id`),
    KEY `idx_child_id` (`child_id`),
    KEY `idx_game_id` (`game_id`),
    KEY `idx_training_date` (`training_date`),
    KEY `idx_child_date` (`child_id`, `training_date`),
    KEY `idx_plan_id` (`plan_id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='训练记录表';

-- 示例result_data
-- {"reaction_times": [1.2, 1.5, 1.3], "error_positions": [5, 12], "completion_time": 95}
```
