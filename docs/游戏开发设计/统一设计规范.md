## 1. 设计理念与品牌规范

### 1.1 品牌定位

| 属性 | 描述 |
|------|------|
| **品牌名称** | 专注小达人 / FocusKids |
| **品牌理念** | 科学有趣，让每个孩子都能专注 |
| **目标用户** | 4-12岁儿童及其家长 |
| **核心价值** | 游戏化训练、正向激励、数据驱动 |

### 1.2 设计语言体系

#### 视觉风格

## 4. 家长端数据报告格式

### 4.1 报告数据结构

```typescript
// 周报数据结构
interface WeeklyReport {
  reportId: string;           // 报告ID
  childId: string;             // 儿童ID
  childName: string;           // 儿童姓名
  reportDate: string;          // 报告日期 YYYY-MM-DD
  period: {
    startDate: string;        // 开始日期
    endDate: string;          // 结束日期
  };
  
  // 基础统计
  basicStats: {
    trainingCount: number;    // 训练次数
    totalDuration: number;    // 总时长(秒)
    avgDuration: number;      // 平均时长(秒)
    activeDays: number;       // 训练天数
  };
  
  // 能力指标
  abilityMetrics: {
    avgFocusScore: number;    // 平均专注度
    avgAccuracy: number;      // 平均准确率
    avgScore: number;         // 平均得分
    completionRate: number;   // 完成率
  };
  
  // 游戏分布
  gameDistribution: {
    [gameCode: string]: {
      gameName: string;
      count: number;
      avgScore: number;
      bestScore: number;
    };
  };
  
  // 趋势数据
  trendData: {
    dates: string[];         // 日期列表
    focusScores: number[];    // 专注度趋势
    accuracyRates: number[];  // 准确率趋势
    durations: number[];      // 时长趋势
  };
  
  // 与上周对比
  comparison: {
    trainingCountChange: number;    // 训练次数变化(%)
    durationChange: number;         // 时长变化(%)
    focusScoreChange: number;       // 专注度变化(%)
    accuracyChange: number;         // 准确率变化(%)
  };
  
  // 亮点与建议
  highlights: Highlight[];    // 亮点
  suggestions: Suggestion[];  // 建议
  generatedAt: string;       // 生成时间
}

// 亮点数据结构
interface Highlight {
  type: 'improvement' | 'achievement' | 'streak';
  title: string;
  description: string;
  value?: number;
  icon: string;
}

// 建议数据结构
interface Suggestion {
  type: 'game' | 'time' | 'method';
  title: string;
  description: string;
  gameCode?: string;
  action?: string;
}
```

### 5.2 同步策略

```
┌─────────────────────────────────────────────────────────────┐
│                    数据同步策略                             │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  实时同步 (LocalStorage)                                    │
│  ├── 游戏过程中的数据 → 立即保存本地                        │
│  ├── 触摸操作、分数变化 → 每步更新                          │

│                                                             │
│  定期同步 (Server)                                          │
│  ├── 游戏结束 → 上报完整数据                                │
│  ├── 成就判定 → 服务端计算                                  │
│  └── 跨设备同步 → 获取最新进度                              │
│                                                             │
│  冲突处理                                                   │
│  ├── 以服务器数据为准                                        │
│  ├── 保留本地日志用于审计                                    │
│  └── 关键操作记录操作日志                                    │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

#### 音量规范
| 类型 | 音量 | 说明 |
|------|------|------|
| BGM | 30% | 背景音乐不干扰 |
| 音效 | 60% | 清晰可闻 |
| 语音 | 80% | 语音指令清晰 |

### 7.2 动画规范

#### 动画时长规范
| 类型 | 时长 | 缓动函数 |
|------|------|----------|
| 微交互 | 100-200ms | ease-out |
| 状态切换 | 200-300ms | ease-in-out |
| 页面切换 | 300-400ms | ease-in-out |
| 复杂动画 | 400-600ms | 自定义 |
| 庆祝动画 | 1000-2000ms | 自定义 |

#### 常用缓动函数
```
ease-out: cubic-bezier(0, 0, 0.2, 1)     - 快速进入，缓慢停止
ease-in: cubic-bezier(0.4, 0, 1, 1)       - 缓慢进入，快速停止
ease-in-out: cubic-bezier(0.4, 0, 0.2, 1) - 两端缓慢，中间快速
spring: cubic-bezier(0.175, 0.885, 0.32, 1.275) - 弹跳效果
bounce: 自定义弹簧函数                  - 成就解锁
```

#### 动画性能规范
```
帧率目标: 60fps
动画策略:
├── 位置/尺寸 → 使用transform
├── 颜色/透明度 → 使用opacity
├── 阴影 → 避免在动画中修改
└── 复杂动画 → 使用Lottie/Spine
```
